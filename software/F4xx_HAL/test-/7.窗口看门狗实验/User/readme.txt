  * 实验现象
  * 1 先点亮LED0延时 300ms后，初始化窗口看门狗，进入死循环，关闭 LED0。
     然后等待窗口看门狗中断的到来，在中断里面，喂狗，并执行 LED1的翻转操作。
     通过 LED0来指示 STM32是否被复位了，如果被复位了就会点亮 300ms。
     LED1用来指示中断喂狗，每次中断喂狗翻转一次。
     所以可以看到LED1不停的闪烁，而LED0只在刚启动的时候闪一下。
